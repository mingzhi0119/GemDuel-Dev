# Cost Token ImageGen Prompt

This prompt produced the 2026-04-23 round CostTokens source sheet. The active
source sheet is kept at `assets/card/overlays/CostTokens/cost-token.png`.

```text
Use case: stylized-concept
Asset type: source sheet for game UI CostTokens, later cropped into transparent PNGs
Primary request: Generate exactly seven isolated circular fantasy board-game cost tokens on a pure chroma green background (#00FF00), arranged in one straight horizontal row with equal spacing. No text, no numbers, no labels.
Subject: Each token is a perfectly centered true circle: a thin polished metallic gold outer rim, a clear colored inner circular field, and subtle decorative floral/damask texture inside the colored field. The gold border must be slim and elegant, about 6-8% of token diameter, not chunky. The inner circle should be large.
Order left to right: red, blue, green, black, white, pearl pink, gold.
Style/medium: high-quality fantasy board-game UI asset, clean icon render, front-facing orthographic view, symmetrical, crisp alpha-like edges, suitable for cutout and Figma overlay use.
Color palette: red is deep ruby; blue is sapphire; green is emerald; black is glossy obsidian; white is porcelain white; pearl is luminous pale rose pearl-pink with nacre sheen, explicitly not beige, not cream, not ivory, not tan; gold is warm metallic gold distinct from the thin rim.
Composition/framing: one row of seven equal-size tokens, each fully visible, centered in its own invisible cell, enough green padding between tokens, no cropping.
Lighting/mood: soft studio lighting, subtle highlights, not overexposed.
Constraints: no right-side small circle, no side badge, no gem mount, no secondary dot, no oval, no perspective tilt, no text, no numbers, no icons, no watermarks, no background scene, no shadows crossing between tokens, no thick gold border, no chunky ornamentation.
Avoid: beige pearl, cream pearl, ivory pearl, tan pearl, extra circles, off-center shapes, duplicate or missing tokens, labels, letters, noisy backgrounds.
```
