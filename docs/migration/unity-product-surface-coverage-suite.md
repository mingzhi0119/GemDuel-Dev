# Unity Product Surface Coverage Suite

Status: complete Local PVP product-surface coverage for the 2026-05-14 migration objective.

This suite turns the declared Unity replacement surface into a finite, machine-checkable coverage model. It is intentionally stricter than seed smoke evidence: the final report can only say `Complete` when every required entrypoint, UI command family, FSM edge, recovery case, settings tuple, and release-runtime packaging requirement has built Windows player evidence.

## Files

- `tools/migration/unity-product-surface-coverage-model.json` freezes the declared finite product surface.
- `tools/migration/run-unity-product-surface-coverage.mjs` collects retained launcher reports, can run fresh built-player scenarios, and writes JSON/HTML coverage reports with a prompt-to-artifact completion audit.
- `tools/migration/run-unity-product-surface-coverage.test.ts` checks that the report only returns `Complete` for a fully covered declared model and fails closed while required surfaces remain uncovered.

## Commands

Run retained evidence only:

```powershell
pnpm unity:product-surface:coverage
```

Run fresh built Windows player scenarios and merge retained evidence:

```powershell
pnpm unity:product-surface:coverage:run
```

Fail unless the report is complete:

```powershell
pnpm unity:product-surface:coverage:check
```

The check command is expected to pass only when the retained and/or freshly generated evidence
reaches `Complete`.

## Artifacts

The default report outputs are:

- `artifacts/unity/product-surface-coverage/unity-product-surface-coverage-report.json`
- `artifacts/unity/product-surface-coverage/unity-product-surface-coverage-report.html`

Every coverage row keeps evidence pointers, replay hashes where present, and missing or blocked reasons. The JSON report also includes `completionAudit.promptToArtifactChecklist`, which maps the numbered product-surface requirements and completion criteria to concrete artifacts and remaining gaps. Launcher reports generated by `coverage:run` are retained next to the report under `artifacts/unity/product-surface-coverage/`.
