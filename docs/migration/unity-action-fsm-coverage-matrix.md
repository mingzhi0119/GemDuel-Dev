## Coverage Conclusion

The matrix is mapped, but full migration completion is blocked. Required player-facing actions
without full valid and rejection coverage include `INITIATE_BUY_JOKER`, `INITIATE_RESERVE`,
`CANCEL_RESERVE`, `RESERVE_CARD`, `RESERVE_DECK`, `DISCARD_RESERVED`, `ACTIVATE_PRIVILEGE`,
`USE_PRIVILEGE`, `CANCEL_PRIVILEGE`, `UNDO`, `REDO`, `PEEK_DECK`, `REROLL_DRAFT_POOL`, and
`CLOSE_MODAL`. The golden corpus now includes 11 fixtures and verifier-enforced `joker-buy`,
`reserved-buy`, `reserve-cancel`, `reserve-deck`, `discard-reserved`, `privilege`, `peek-modal`,
`draft-reroll`, `draft-p2-reroll`, and `royal-handoff` tags. The verifier now also reads
`fixtures/replay-golden/rejection-manifest.json`, which locks 54 wrong-phase, resource, ownership,
mismatch, follow-up, modal, and derived-state rejection cases to expected reasons and unchanged
replay-state hashes. Unity EditMode now replays all 54 committed rejection-manifest cases by
priming each revision from the TypeScript replay-state oracle, sending the invalid action through
the live `IGameRulesEngine` bridge, and checking unchanged replay-state hashes plus unchanged live
recording counts. Replay simulation now filters candidate actions that would leave the Replay
vNext state hash unchanged before recording them into golden output. The TypeScript bridge also
applies every golden replay event as commands and hash-checks each resulting state without replacing
checkpoint snapshots. It now also drives one freshly simulated deterministic Local PvP replay from
bridge start to game over through the same command boundary with per-step hash checks. Unity
EditMode additionally drives three seeded fresh product-surface Local PvP games from semantic start
to game over, checks each live recording hash, exports Replay vNext JSON, imports each replay into
a separate controller, and reviews to the same final hash. These fixtures, bridge proofs, and
seeded product-surface game proofs still do not yet satisfy the broader actor-ordering, release-path,
online, undo/redo release-path, remaining edge, and full arbitrary product UI requirements needed
for a full migration completion claim.
Reserved-card buy now has a bridge unit proof plus live Unity preview dispatch and affordable
recording/review proof. Joker market buy now has a bridge unit proof plus live Unity
color-selection dispatch, affordable recording/review proof, and pending LocalDev recovery proof.
Discard-reserved now has golden fixture coverage plus bridge and live Unity preview dispatch proof,
draft reroll now has P1 and P2 ordered golden fixture coverage plus bridge and live Unity target
proof, deck peek/modal close have golden fixture coverage plus bridge and live Unity replay-review proof, reserve deck plus
reserve cancel have golden fixture coverage plus bridge and live Unity follow-up proof, royal
selection now has explicit next-player handoff plus extra-turn oracle coverage, and
bonus/discard/steal follow-up rejection paths have bridge plus live Unity no-mutation proof.
