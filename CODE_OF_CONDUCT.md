# Code of Conduct

## Standard

GemDuel-Dev contributors are expected to keep technical discussion respectful, specific, and focused on the work. Review comments should describe observable behavior, risk, and suggested changes without personal attacks.

## Scope

This code of conduct applies to repository issues, pull requests, code review, release discussions, and project documentation.

## Reporting

Report conduct problems privately to the repository owner. Security issues should use the private GitHub advisory URL documented in `SECURITY.md` instead of this process.

## Enforcement

Maintainers may edit or remove abusive content, pause a discussion, or block participation when behavior disrupts safe project work. Enforcement should be proportional, documented when practical, and separated from technical disagreement.
